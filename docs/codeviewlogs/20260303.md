🔴 高风险（必须修复）
[auth.py:34 vs user_manager.py:43] JWT 密钥不一致
auth.py 的 _get_jwt_secret() 默认值是 "agenticx-dev-secret-change-in-production"，而 user_manager.py 的默认值是 "agenticx-dev-secret"。两处读取同一个环境变量 AGENTICX_JWT_SECRET，但 fallback 不同。
影响：当 AGENTICX_JWT_SECRET 未设置时（开发环境），user_manager.generate_jwt() 用 "agenticx-dev-secret" 签名，而 JWTAuthMiddleware 用 "agenticx-dev-secret-change-in-production" 验证 —— 所有 JWT 验证必然失败。
修复：统一为同一个常量或同一个函数。
# 建议：auth.py 和 user_manager.py 共用同一个 getter_DEFAULT_JWT_SECRET = "agenticx-dev-secret-change-in-production"def _get_jwt_secret() -> str:    return os.environ.get("AGENTICX_JWT_SECRET", _DEFAULT_JWT_SECRET)
[middleware.py:180-207] 中间件注册顺序导致 RateLimit 无法获取 user_id/tenant_id
Starlette 中间件执行顺序是 LIFO（最后注册的最先执行）。当前注册顺序：
RequestId → JWT → Tenant → Timeout → RateLimit → CircuitBreaker
实际执行顺序变为：CircuitBreaker → RateLimit → Timeout → Tenant → JWT → RequestId
这意味着 RateLimitMiddleware 在 JWT/Tenant 之前执行，request.state.user_id 和 request.state.tenant_id 尚未设置，限流永远回退到 "anonymous"/"default"。
影响：per-user / per-tenant 限流策略完全失效，所有请求共享同一个匿名桶。
修复：反转注册顺序，使最先执行的中间件最后注册：
def register_production_middlewares(app, config=None):    cfg = config or MiddlewareConfig()    # 注册顺序 = 执行顺序的反序（LIFO）    if cfg.enable_circuit_breaker:        app.add_middleware(CircuitBreakerMiddleware, ...)    if cfg.enable_rate_limit:        app.add_middleware(RateLimitMiddleware, ...)    if cfg.enable_timeout:        app.add_middleware(TimeoutMiddleware, ...)    if cfg.enable_tenant_isolation:        app.add_middleware(TenantIsolationMiddleware)    if cfg.enable_jwt_auth:        app.add_middleware(JWTAuthMiddleware)    if cfg.enable_request_id:        app.add_middleware(RequestIdMiddleware)
[tenant.py:63-64] X-Tenant-ID header 可被未认证请求伪造
TenantIsolationMiddleware 在 JWT 验证之前（由于上述 LIFO 问题更严重），允许任意客户端通过 X-Tenant-ID header 设置任意 tenant_id。
影响：跨租户数据访问 —— 未认证用户可以设置 X-Tenant-ID: victim-tenant 访问其他租户数据。
修复：仅信任 JWT/API-Key 中的 tenant_id，将 header 降级为最低优先级或仅在开发模式下启用：
if hasattr(request.state, "auth") and request.state.auth.authenticated:    tenant_id = request.state.auth.tenant_idelif os.environ.get("AGENTICX_DEV_MODE") and request.headers.get("X-Tenant-ID"):    tenant_id = request.headers.get("X-Tenant-ID")
[server.py:164 vs api_routes.py:224] 重复注册 /health 路由
server.py 的 _register_routes 注册了 GET /health 返回 {"status": "healthy"}，而 api_routes.py 也注册了同路径返回 HealthResponse().dict()。
影响：路由冲突，行为取决于注册顺序，可能导致健康检查返回不一致的格式。
修复：移除 server.py 中的静态 /health 路由，统一使用 api_routes.py 中带有 HealthProbe 的版本。

[auth.py:37-45] AuthState 使用类级别可变默认值
class AuthState:    roles: List[str] = []    permissions: List[str] = []
类级别的 [] 是共享对象。虽然当前代码总是通过赋值创建实例属性（安全），但这是一个脆弱的设计 —— 未来任何 auth.state.roles.append(...) 都会污染所有实例。

[task_queue.py:71] tasks 字典无界增长
AsyncTaskQueue._tasks 只增不减，完成/失败的任务永不清理。长时间运行的服务会持续泄漏内存。
建议：增加 TTL 清理或 max_history 限制：
async def _cleanup_old_tasks(self, max_age_seconds=3600):    now = time.time()    async with self._lock:        to_delete = [            tid for tid, info in self._tasks.items()            if info.completed_at and now - info.completed_at > max_age_seconds        ]        for tid in to_delete:            del self._tasks[tid]
[resilience.py:146-152] 幂等性保护存在竞态窗口
当两个并发请求携带相同 Idempotency-Key 时：
请求 A 先到，set_if_absent 成功，开始执行
请求 B 到达，set_if_absent 失败，sleep 0.1s 后检查
如果请求 A 在 0.1s 内未完成，请求 B 读到 _PENDING，但代码没有处理这个情况 —— cached is not _PENDING 判断为 False，请求 B 直接 fallthrough 到 retry 循环，重复执行
建议：增加重试等待循环直到结果就绪或超时。
[middleware.py:143] CircuitBreaker 字典无界增长
_breakers 为每个 METHOD:PATH 组合创建 CircuitBreaker。虽然 FastAPI 路由路径有限，但中间件在路由匹配之前执行 —— 扫描器可以用随机路径触发创建。
建议：增加 LRU 淘汰或仅对已注册路由创建 breaker。
[api_routes.py:399-429] /api/login-by_stack 端点无真实认证
返回硬编码用户信息（stack_user, stack@example.com），无任何 token 验证。
建议：标记为 deprecated 或增加实际验证逻辑，否则生产环境应禁用。
[health.py:52-113] DependencyChecker 三个检查方法重复代码
check_database(), check_llm_provider(), check_memory_backend() 逻辑完全相同，仅名称不同。
建议：提取通用方法：
async def _check(self, name: str, checker: Optional[Callable]) -> CheckResult:    if not checker:        return CheckResult(name, HealthStatus.UNKNOWN, "No checker configured")    start = time.perf_counter()    try:        result = checker()        if asyncio.iscoroutine(result):            result = await result        return CheckResult(name, HealthStatus.HEALTHY, "OK", latency_ms=(time.perf_counter()-start)*1000)    except Exception as e:        return CheckResult(name, HealthStatus.UNHEALTHY, str(e), latency_ms=(time.perf_counter()-start)*1000)
