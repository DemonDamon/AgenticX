# Memory UX P0: Heading-Aware Chunking + Favorites-to-Recall Pipeline

Plan-Id: 2026-03-23-memory-ux-p0

## Background

AgenticX 的 `WorkspaceMemoryStore` 当前使用固定 40 行分块策略（`_chunk_text`），导致：

- MEMORY.md 中按 `##` 组织的条目可能被切断跨两个 chunk
- FTS 和语义搜索召回精度低（实测 semantic score 仅 0.2x）
- 用户显式 "记住" 的信息无法被精确命中

同时，Desktop "收藏" 按钮只写入 `scratchpad["saved_messages"]`，完全不进入 `WorkspaceMemoryStore` 索引，用户收藏的重要消息在后续对话中无法被自动召回。

## T1: Markdown Heading-Aware Chunking

**File**: `[agenticx/memory/workspace_memory.py](agenticx/memory/workspace_memory.py)` — `_chunk_text` method (lines 223-235)

**Current behavior**:

```python
def _chunk_text(self, content: str) -> Iterable[Tuple[int, int, str]]:
    lines = content.splitlines()
    chunk_size = 40
    for start in range(0, len(lines), chunk_size):
        end = min(len(lines), start + chunk_size)
        text = "\n".join(lines[start:end]).strip()
        if text:
            out.append((start + 1, end, text))
```

**New behavior**:

1. 扫描所有以 `#`  / `##`  / `###`  等开头的行作为分界点
2. 每个 section（heading + 其后内容直到下一个 heading）作为一个 chunk
3. 如果某个 section 超过 `MAX_SECTION_LINES`（60 行），在段落空行处二次切分
4. 如果文件中无 heading，回退到当前 40 行策略
5. 空 chunk 跳过

**Implementation sketch**:

```python
_HEADING_RE = re.compile(r"^#{1,6}\s")
MAX_SECTION_LINES = 60
FALLBACK_CHUNK_SIZE = 40

def _chunk_text(self, content: str) -> Iterable[Tuple[int, int, str]]:
    lines = content.splitlines()
    if not lines:
        return []
    
    # Find heading positions
    heading_indices = [i for i, line in enumerate(lines) if self._HEADING_RE.match(line)]
    
    if not heading_indices:
        # Fallback: fixed-size chunks
        return self._chunk_fixed(lines)
    
    sections = []
    for idx, start in enumerate(heading_indices):
        end = heading_indices[idx + 1] if idx + 1 < len(heading_indices) else len(lines)
        sections.append((start, end))
    
    # Include any content before the first heading as a preamble chunk
    if heading_indices[0] > 0:
        sections.insert(0, (0, heading_indices[0]))
    
    out = []
    for start, end in sections:
        if end - start <= MAX_SECTION_LINES:
            text = "\n".join(lines[start:end]).strip()
            if text:
                out.append((start + 1, end, text))
        else:
            # Sub-split large sections at paragraph boundaries
            out.extend(self._subsplit_section(lines, start, end))
    return out
```

**Why this improves UX**:

- "## 火山方舟 Coding Plan 入口" 及其下方 URL/说明保持在同一个 chunk
- FTS MATCH "火山方舟" 直接命中整个条目，不会只返回半截
- Semantic embedding 覆盖完整语义单元，相似度分数显著提升

## T2: 收藏消息写入长期记忆

**File**: `[agenticx/studio/server.py](agenticx/studio/server.py)` — `save_message_memory` endpoint (lines 1748-1776)

**Current behavior**: 只写 `scratchpad["saved_messages"]`，不进入 `WorkspaceMemoryStore`。

**New behavior**:

1. 保留现有 scratchpad 逻辑不变（向后兼容）
2. 新增：截取内容前 500 字符（避免超长助手消息膨胀 MEMORY.md），加 `## 用户收藏` 前缀
3. 调用 `append_long_term_memory(workspace_dir, formatted_content)` 写入 MEMORY.md
4. 调用 `WorkspaceMemoryStore().index_workspace_sync(workspace_dir)` 刷新索引
5. 返回 response 增加 `"memory_persisted": True`

**Implementation sketch** (在 `save_message_memory` 函数尾部追加):

```python
from agenticx.workspace.loader import append_long_term_memory, resolve_workspace_dir
from agenticx.memory.workspace_memory import WorkspaceMemoryStore

# ... existing scratchpad logic ...

# Persist to long-term memory for recall
memory_persisted = False
try:
    workspace_dir = resolve_workspace_dir()
    truncated = content[:500].strip()
    append_long_term_memory(workspace_dir, f"[用户收藏] {truncated}")
    WorkspaceMemoryStore().index_workspace_sync(workspace_dir)
    memory_persisted = True
except Exception:
    pass  # best-effort, don't block the API response

return {"ok": True, "saved_count": len(scratch["saved_messages"]), "memory_persisted": memory_persisted}
```

**Why this improves UX**:

- 用户收藏的重要消息在下次对话时可被 `_build_memory_recall_context` 自动召回
- 无需用户手动在对话中说 "记住这个"
- 收藏按钮从 "私人书签" 升级为 "跨会话记忆锚点"
