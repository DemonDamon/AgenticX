# AGENTS.md

## Learned User Preferences
- 默认使用中文回复；技术术语可按需保留英文。
- 进行 git commit 时，提交信息必须包含 `Made-with: Damon Li`，并偏好按功能点分组、附结构化需求块（如 FR/NFR/AC）。
- Plan 文档必须落盘到 `.cursor/plans/` 并随代码一起提交，不能遗漏。
- Desktop 端视觉重塑：App 命名为「Machi」，应用图标偏好《全职猎人》玛奇神韵的极客化解构——纯黑白高对比度矢量线稿（类 NousResearch 风格），仅保留至颈部的大头贴（无肩/无头巾），以高马尾和冷酷洞悉的眼神凸显“绝对理性”的高级开发者工具气质，拒绝低端二次元感或渐变色。
- 配置面板遵循关注分离：MCP 独立 tab 不混入 Provider；用户可改配置项必须提供 Desktop 设置面板 GUI；模型切换需持久化；权限确认文案需对齐 Cursor，且用户选择 `Run Everything` 后必须严格生效，禁止重复弹窗询问。主要操作按钮（如保存）应使用主题层 `--ui-btn-primary-*` 变量并按 dark/dim/light 区分，避免长期硬编码 cyan。配置面板 UX 持续以 Cherry Studio 为参照：秘钥记录与校验、模型列表管理与健康检查、@切换模型回答等交互。
- 多分身/群聊 UX 深度对齐微信体验：支持 @提及指定分身直接回复、未 @ 时由 Meta-Agent（产品名 Machi、显示名以 Meta 窗格为准）充当项目经理统筹/兜底；未 @ 但文本明显指向某成员职责时，路由应优先由该成员实际执行与回复，而非仅靠 Meta 口头代答。用户点名某分身时，该分身应以人类用户为主答对象，避免把主对话改成 @ 组长客套；成员在回复里 @ 其他分身时，路由应尽力让被 @ 方接续发言。用户需要能感知分身在执行（工具调用、阻塞确认、进度类信号），避免长时间只有「正在输入」却看不到在做什么。新建群聊必须默认隐式包含 Meta-Agent；消息多选需左侧打勾，多选复制须结构化（角色+时间+内容），合并转发卡片点开需见原始历史。Session 创建参考 Cherry Studio；默认布局需视觉均衡，窄屏头像自适应。设置里「群内显示名称」参与群聊上下文标注（`user_display_name`）。
- UI 交互必须即时响应（乐观 UI）：打开窗格、删除分身/群聊、切换会话都要先响应再异步回填；删除群聊后应回退到元 Agent 界面而非停留在已删除群聊；转发卡片等消息体必须自适应大小避免内容遮挡；工作区面板布局自上而下：成员区 → 工作目录 → Spawn Agent 列表（成员不单独 tab）；消息操作按钮（复制/引用/收藏/转发/重试/多选）应常驻显示而非仅 hover 出现，且用户消息行固定提供重试（非仅失败时才出现）；收藏结果反馈（如「已收藏过」）须贴近该条消息下方操作按钮行展示，避免仅在侧栏或会话列表远端角落弱提示；引用必须优先使用用户选区文本（不要整段带上 `<think>`）；多选操作必须包含删除且能持久化生效；流式回答禁止出现整段重复拼接；流式输出期间不得强制锁定滚动到底部，用户需能随时向上滚动查看已输出内容；附件上传后须在输入框**上方**展示预览（对齐豆包/Cherry Studio），图片显示可点击放大的缩略图；重试或复制消息时必须保留图片附件，不得退化成纯文本；删除所有会话后须自动新建默认 session 而非卡在「正在初始化」。
- 子智能体须全链路透明（运行状态/摘要/产出/`awaiting_confirm` 与倒计时）；完成或失败后 Meta-Agent 要主动汇报；代码生成须有类似 Cursor 的流式呈现，等待期间不得只显示 ⏳⏳ 静态符号，应使用类豆包的动态三点动画。委派须为「真委派」：任务在分身真实 session 执行、历史与产出可追溯，禁止影子 spawn 顶替身份，启动后自动打开目标分身窗格；子智能体详情面板须有一键复制按钮。Desktop 端应支持「断点续开」：合盖/重开后优先恢复上次会话与多窗格状态，避免用户重新找上下文。
- LLM `<think>` 推理内容必须解析为独立 ReasoningBlock：streaming 阶段 spinner + "Thinking" 标签 + 流式推理文本在同一容器内；完成后折叠为 "Thought"；禁用独立边框/白色背景/青蓝色底色——融入消息气泡（参照 Cursor 思考链 UI）。
- 用户对重复噪音提示敏感：须收敛为单次高信号、降低焦虑感；已知非视觉聊天模型下用户尝试附图时，用 Cherry 式文案「模型不支持该文件类型」等明确提示，展示在**消息列表与输入区之间的主视区**内水平居中（紧邻状态 chips、贴近输入区上方），采用黄色感叹号警示样式，避免缩在窗口右下角或边角弱提示；内部系统消息（如工具调用频率限制日志）对用户不可见。UI 文案区分语义：操作仅取消关联/隐藏时用「移除」，真正删除文件/数据时才用「删除」，避免引起用户不必要的担心。
- 出现线上异常时优先做根因排查和证据链，不接受拍脑袋修复；实现新需求时严禁乱改已有逻辑正确的代码（用户将此称为「严重倒退现象」）；已有 workspace rule `no-scope-creep.mdc` 强制约束，每次改动必须能追溯到具体需求。git commit 必须精准：只 `git add` 本次任务直接改动的文件，绝不把无关已修改文件一起提交。
- Desktop toolbar 按钮图标化规范：工作区→文件夹图标（tab 仅显示图标，hover tooltip 显示名称）、Spawns→机器人图标、历史→聊天气泡图标（不用时钟）；toolbar 按钮图标须与对应面板 header 图标一致；历史面板 active 状态须用 `bg-surface-card-strong text-text-strong`，与工作区/Spawns 按钮激活样式对齐。
- 技术笔记和博客偏好"图文并茂"风格，常落盘到飞书文档库（指定文档空间下）；回复 GitHub Issue 时要求简约、非 AI 化、英文版。

## Learned Workspace Facts
- AgenticX 全局配置目录为 `~/.agenticx`（主配置 `~/.agenticx/config.yaml`），身份与记忆目录为 `~/.agenticx/workspace`；`AGX_MAX_TOOL_ROUNDS` 等运行参数可在该文件配置。长期记忆应写入该 workspace（如 `MEMORY.md`、`memory/*.md`）并进入 `WorkspaceMemoryStore` 索引。首次启动的 Pro/Lite 模式选择由 `onboarding_completed` 写入该配置文件控制，未完成或配置重置/新环境会再次显示引导。
- 元智能体系统提示由 `agenticx/runtime/prompts/meta_agent.py` 构建，每轮动态注入活跃子智能体快照（`_build_active_subagents_context`）与记忆召回（`_build_memory_recall_context`）。
- 子智能体调度由 `agenticx/runtime/team_manager.py`（`AgentTeamManager`）管理，支持并发上限、归档快照、`owner_session_id` 会话隔离、`avatar_id` 绑定与跨 session 状态查询。
- `agenticx/runtime/agent_runtime.py` 会清洗上下文并强制 tool 调用序列合法，避免 `assistant(tool_calls)` / `tool` 断链导致 provider 400；streaming 路径会归一化 `None` 工具名；`[系统通知]` 不写入 `chat_history`。子智能体侧无意义重复工具调用由 `agenticx/runtime/loop_detector.py` 检测（含 tool saturation 等与 `agent_runtime` 协同）。
- `agenticx/studio/server.py` 是纯 REST API + SSE 后端（无内置 Web UI），子智能体状态/取消/重试接口支持同会话 fallback；`chat_history` 持久化到 `messages.json` 并由 `GET /api/session/messages` 暴露；群聊与普通会话需传递 `quoted_content`；批量删除消息需同步持久化。
- `agenticx/runtime/meta_tools.py` 中 `spawn_subagent` 会拦截已注册 avatar 名称并引导 `delegate_to_avatar`；`delegate_to_avatar` 通过 `_find_or_create_avatar_session` + `_run_delegation_in_avatar_session` 在分身会话后台执行，provider/model 按 avatar → session → Meta-Agent 回退；`/api/subagents/status` 扫描 `_delegation_info`，委派后台任务约每 5 秒中间持久化供前端轮询。`spawn_subagent` 的 `run_timeout_seconds` 若低于默认 600s 会覆盖默认，长任务易误超时；可用环境变量 `AGX_SUBAGENT_MIN_RUN_TIMEOUT_SECONDS` 调整运行时下限（设为 `0` 可关闭抬高短超时的保护）。
- Desktop 技术栈为 React + Zustand + Electron + Vite（Pro/Lite：`ChatPane` / `ChatView`）；`store.ts` 含 `awaiting_confirm`；推理块链路需保持 `reasoning-parser.ts`、`ReasoningBlock.tsx`、`ImBubble`/`CleanBlock`/`TerminalLine` 一致。DMG 自包含打包方案：`packaging/pyinstaller/agx_serve_entry.py` + `agx_serve.spec`（`collect_all("agenticx")`，非 editable 安装）+ `packaging/build_backend.sh`（venv 隔离 + `curl --noproxy` smoke 验证）；已有 plan `.cursor/plans/2026-03-24-dmg-self-contained-packaging.plan.md`。DeerFlow 2.0 codegen 已落地三个核心机制：Guardrails（`agenticx/tools/guardrails/`，含 `GuardrailProvider` Protocol + `DenyListGuardrail`）、MCP sync（`agenticx/tools/mcp_sync_tool.py`）、Token SSE（`agenticx/runtime/usage_metadata.py`）；对应 plan `.cursor/plans/2026-03-25-deerflow-2-codegen.plan.md`。Desktop 集成终端依赖 `node-pty`（native module），每次 Electron 版本升级后必须重跑 `npx @electron/rebuild -f -w node-pty`；`checkAgxCli()` 超时须设 ≥30s（Python 冷启动实测 ~9s）；pty spawn 用 `-i` flag 并备用路径 chain（`$SHELL` → `/bin/zsh` → `/bin/bash`）。Machi Electron 主进程须 `app.requestSingleInstanceLock()` 与 `second-instance` 聚焦已有窗口，避免 macOS 多进程双窗；`BrowserWindow` 使用 `show: false` 并在 `ready-to-show` 后再显示；打包 `loadFile` 失败须内嵌错误页而非静默吞错。修改 `desktop/electron/main.ts` 或新增 IPC 后须重新编译进 `dist-electron/` 并完全退出再启动（⌘Q / 重启 `npm run dev`），仅刷新渲染进程不会加载新 IPC handler。工作区面板内嵌终端区域须用主题 token（如 `bg-surface-card`），禁止硬编码 `bg-[#0f1014]`；`@xterm/xterm` 默认 `.xterm-viewport { background: #000 }` 在 `desktop/src/index.css` 覆写为透明；`TerminalEmbed` 用面板计算背景色与 `getTerminalAnsi(theme)` 对齐 light/dim/dark。
- 分身由 `agenticx/avatar/registry.py` 与 `group_chat.py` 管理；群聊默认注入 Meta-Agent；`group_router.py` 负责 `user-directed` / `meta-routed` 路由与 mention 归一化，支持成员间 `@` 触发后续接话。文件类工具（如 `file_read`）解析路径须与会话/用户配置的工作区根一致，不应仅在分身默认目录 `~/.agenticx/avatars/<avatar_id>/workspace` 下查找用户已加入工作区或通过 `@` 引用的仓库文件。
- `agenticx/llms/minimax_provider.py` 用 `@model_validator(mode="after")` 自动补 `openai/` 前缀；MiniMax M2 系列按非视觉模型处理，前端需拦截不支持的图片输入，Studio 需剥离 `image_inputs`。智谱 `glm-5` / `glm-5-*`（名称不含 vl、vision、4v、5v）按纯文本模型处理，Studio 同样剥离 `image_inputs`，Desktop 在 `model-vision.ts` 拦截附图。
- 扩展生态位于 `agenticx/extensions/`：`bundle.py` 定义 AGX Bundle（skills/mcp_servers/avatars/memory_templates），`installer.py` 负责本地安装卸载，`registry_hub.py` 做多源聚合搜索；Studio 暴露 `/api/skills/*`、`/api/bundles/*`、`/api/registry/*`。Desktop 在 Windows 平台默认禁用 GPU 合成（`disable-gpu` + `disableHardwareAcceleration`）以规避重影与拖影；非 Windows 仍可用 `AGX_DISABLE_GPU=1` 强制禁用；Windows 安装时 `import resource` 模块不兼容，需条件导入。Computer Use 能力内化于 `agenticx/tools/fallback_chain.py`（`ToolFallbackChain`）、`agenticx/tools/resolvers/`（`MCPConnectorResolver`、`ComputerUseResolver`）、`agenticx/tools/policy.py`（`CategoryPolicy`）、`agenticx/embodiment/tools/desktop_adapter.py`（`DesktopPlatformAdapter`）、`agenticx/runtime/task_scheduler.py`；已有 plan `.cursor/plans/2026-03-24-computer-use-internalization.plan.md`。通过 MCP 接入的 browser-use 类浏览器工具在实战中出现已连接仍 `mcp_call` 失败、`ClosedResourceError`、参数校验错误或与不兼容的浏览器上下文 API 并存的情况，智能体可能回退到 bash/Playwright 等直连路径，排障时需对照连接状态、tool schema 与实际调用链。Skill 采用双路径注入：路径 1 为 Tool Result + 三层渐进披露，路径 2 为 System Prompt 注入（`active_skill`）；框架层推荐路径 1。官网 `https://www.agxbuilder.com/`。
- browser-use MCP：`browser_extract_content` 在 MCP 子进程内需可用的 `OPENAI_API_KEY`（及与 browser-use 兼容的 `OPENAI_BASE_URL`/`OPENAI_MODEL`，纯文本模型即可）；会话删除或过期清理应关闭该 session 的 `mcp_hub`，否则易残留多个 `browser-use` stdio 进程争抢 CDP，表现为 `browser_get_state`/`browser_get_html` 失败或会话列表异常。browser-use 已从 auto-connect 中彻底排除（所有会话均不自动连接），需要时须显式调用 `mcp_connect`；用 `--profile Default` 可复用 Chrome 登录 Cookie，但操作前须先完全关闭 Chrome，否则 CDP 报"浏览器已被占用"。
- Desktop 当前硬绑本地后端（`spawnAgx(["serve", "--host", "127.0.0.1", ...])`），不支持连接远程 `agx serve`；`agx serve` CLI 支持 `--host`/`--port`/`--reload`，CORS 白名单硬编码 `localhost:5173` 系列；已有 plan `.cursor/plans/2026-03-24-desktop-remote-backend.plan.md` 规划远程模式支持。
- GitHub Actions 构建 Machi 桌面（`.github/workflows/build-desktop.yml`）：`jobs.<job>.if` 中不可引用 `matrix`（需拆 job）；在单台 macOS runner 上无法可靠对 `agx-server` 做 arm64+x64 的 `lipo` universal；electron-builder 25.x 要求 `desktop/package.json` 含 `author`；CI 中 `electron-builder` 宜加 `--publish never`；无 Apple 签名证书时不向 env 传空的 `APPLE_*`/`CSC_*`，并设 `CSC_IDENTITY_AUTO_DISCOVERY=false`，避免在未签名构建里启用 `hardenedRuntime`/entitlements 或误配 `afterSign` 触发签名校验；推送含 workflow 变更需 PAT 含 `workflow` scope。Runner 使用 Python 3.12 时标准库已移除 `distutils`，在 `setup-python` 之后应 `pip install setuptools`，否则 `node-gyp` 编译 `node-pty` 可能失败。
